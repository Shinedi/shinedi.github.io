2、tree shaking（摇树优化）

  1. 概念：1个模块可能有多个方法，只要其中的某个方法使用到了，则整个文件都会把打包到bundle里面去，tree shaking 就是只把用到的方法打入bundle,没用到的方法会在uglify阶段被擦除掉
  2. 使用： webpack默认支持，在.babeIrc里设置modules:false即可
    production mode的情况下默认开启
  3. 要求：必须是ES6的语法，CJS（commonJS）的方法不支持
  4. DCE
    代码不会被执行，不可到达
    代码执行的结果不会被用到
    代码只会影响死变量（只写不读）
  5. Tree Shaking原理：利用ES6模块的特点
    只能作为模块顶层的语句出现
    import的模块名只能是字符串常量
    import的模块是不能修改的
    代码擦除：uglify阶段删除无用代码

3、Scope Hoisting使用和原理分析

  1. 现象：构建后的代码存在大量闭包代码
  2. 会导致什么问题？
    大量函数闭包包裹代码，导致体积增大(模块越多越明显)
    运行代码时创建的函数作用域变多，内存开销变大
  3. 模块转换分析
    被webpack转换后的模块会带上一层包裹
    import会被转换成__webpack_require
  4. 进一步分析webpack的模块机制
    打包出来是一个匿名闭包(IIFE)
    modules是一个数组,每一项是一个模块初始化函数
    __webpack_require用来加载模块,返回module.exports
    通过webpack_require_method(0)启动程序
  5. scope hoisting原理
    将所有模块的代码按照引用顺序放在一个函数作用域里,然后适当的重命名一些变量以防止变量名冲突
    对比：通过scope hoisting可以减少函数声明和内存开销
    mode设置为production会自动打开scope hoisting,但是这种方式不方便我们看打包后的代码，所以可以先设置mode为none,再添加下面的plugin

4、代码分割和动态import

  1. 代码分割的意义
    对于大的web应用，将所有代码放在一个文件中显然是不够有效的，特别是当年你的代码是在某些特殊的时候才被使用到。webpack是将你的代码分割成chunks(语块)，当代码运行到他们的时候进行加载
    运行场景：
    抽离相同代码到一个共享块
    脚本懒加载,是的初始下载代码更小

  2. 懒加载脚本的方式
    commonJS: require.ensure
    ES6: 动态import(目前还没有原生支持，需要babel转换)

  3. 如何使用动态import呢
    安装babel插件
    npm install @babel/plugin-syntax-dynamic-import -D
    ES6: 动态import(目前还没有原生支持,需要Babel转换)
