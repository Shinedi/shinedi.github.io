3、冒烟测试介绍和实际应用

  1. 冒烟测试：对提交测试的软件在进行详细深入的测试前而进行的预测试,避免出现基本功能确实等严重问题

  2. 冒烟测试执行
    构建是否成功
    每次构建完成build目录是否有内容输出

  • 是否有js、css等静态资源文件
  • 是否有HTML文件

  1. 判断是否构建成功
    在示例项目里面运行构建,看看是否有报错

  2. 判断基本功能是否正常
    编写测试用例

  • 是否有js、css等静态资源文件
  • 是否有HTML文件
  1. 用法：
    编写测试用例
  • 安装rimraf(可以删除dist目录)：npm i rimraf -D
  • 在test/smoke下创建template目录和index.js,smoke下放入代码，index中创建冒烟测试用例
  • 修改webpack.base.js中的__dirname
  • 运行node test/smoke/index.js
  1. 单元测试用例： 通过冒烟测试后,编写单元测试用例
    (1). 安装抹茶： npm i mocha -D
    安装glob:npm i glob-all -D
    (2). index.js (3). css-js-test.js (4)、html-test.js

4、 单元测试和测试覆盖率

  1. 编写单元测试用例
    (1)、技术选型: Mocha+Chai
    (2)、测试代码：describe(描述用来测试的文件),it（包裹测试用例）,except
    (3)、测试命令：mocha add.test.js

  2. 单元测试接入
    1、安装mocha
    2、新建test目录，添加xxx.test.js
    3、在package.json中的scripts字段增加test命令
    {
    test: ‘mocha’
    }
    4、执行测试命令：npm run test(抹茶会默认找到test下的index.js)
