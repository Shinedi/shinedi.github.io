webpack核心概念

####1、entry
Entry用来指定webpack打包入口

  • 用法

    • 单入口：entry是一个字符串

    module.exports = {

    entry: './path/xsss'

    }

  • 多入口：entry是一个对象

####2、output

output用来告诉webpack如何将编译后的文件输出到磁盘

  • 用法
    • 单入口
    • 多入口(通过占位符确保文件名称统一)

####3、Loaders
webpack开箱只支持JS和JSON两种文件类型,通过loaders去支持其他文件类型并转成有效模块,添加到依赖图中

本身是一个函数,接受源文件作为参数,返回转换的结果

  • loader调用是从右往左链式调用

####4、plugins
作用于整个构建过程，loaders没办法做的事情都可以通过plugins完成，例如文件优化

####5、mode
mode用来指定当前的构建环境是：production、development、none
默认值是production
根据不同环境开启相应优化选项
process.env.NODE_ENV

####6、解析ES6
使用babel-loader,babel-loader是依赖babel的，babel的配置文件是.babeIrc

  • babelrc配置
  • 安装：npm i @babel/core @babel/preset-env babel-loader -D
    （@babel/core是babel的核心模块，-D是–save-dev简称）

####7、解析css、less、sass

  • css-loader用于加载.css文件，并且转换成commonJS对象
  • style-loader将样式通过style标签插入到head中
  • 安装： npm i style-loader css-loader less less-loader
    less-loader是依赖less的
  • webpack.config.js

####8、解析图片和字体

  • file-loader用于处理文件（包括图片和字体）
  • webpack.config.js
  • url-loader也可以处理图片和字体,可以设置较小资源自动base64

####9、webpack中的文件监听
文件监听是在发现源码发生变化时,自动重新构建出新的输出文件

webpack开启监听模式，有两种方式：

  • 启动webpack命令时，带上–watch参数

  • 在配置webpack.config.js中设置watch: true

  • 也可在package.json中添加命令

    唯一缺陷：每次需要手动刷新浏览器

  • 文件监听的原理分析
    轮询判断文件的最后编辑时间是否变化
    某个文件发生了变化，并不会立刻告诉监听者，而是先缓存起来，等aggregateTimeout

####10、热更新及原理分析

  • 热更新：webpack-dev-server
    wds不刷新浏览器
    wds不输出文件，而是放在内存中
    使用HotModuleReplacementPlugin插件
  • package.json(需要安装webpack-dev-server)
    • webpack.config.js
